Wire format examples

Examples

Concrete AHF outputs with annotations. These are generated from the Go reference implementation. Use them to sanity-check your own implementation or to understand the wire format before reading the full spec.

01

AH1 — agent-help index

Source command: mem --agent-help

AH1 is what an agent gets when it runs tool --agent-help at the root. It provides a compact index of all invocable commands. The more? record is a pointer — not a shell command — telling the agent how to get detail on any command.

Raw AHF output

ah1 mem :: project memory — store and query facts, decisions, and tasks
cmd node add <text> --type TYPE [--tag K:V...] :: store a new memory node
cmd node list [--type TYPE] [--limit int] [--cursor id] :: list memory nodes
cmd search query <text> [--type TYPE] [--limit int] [--cursor id] :: search nodes by text
cmd search similar <id> [--limit int] [--cursor id] :: find nodes similar to a given node
cmd project status :: show current project settings
cmd project set --key KEY --value VALUE :: set a project setting
more? mem <cmd> --agent-help

Notes

  • ah1 header: ah1 <tool> :: <purpose>
  • Each cmd line shows the full command path, required args in <angle brackets>, and key optional flags in [square brackets]
  • :: separates the signature from the human-readable purpose
  • more? is a pointer record — the ? is intentional. Do not copy more? into a shell command.
  • Target: fewer than 300 tokens for the entire AH1 output
02

AH2 — command detail

Source command: mem node add --agent-help

AH2 is the per-command detail page. An agent fetches this after seeing a command in AH1. It gives the agent everything needed to construct a correct invocation: the canonical form, typed arguments, typed flags, and working examples.

Raw AHF output

ah2 mem node add
use mem node add <text> --type TYPE [--tag K:V...]
arg text:str req :: node text content
flag --type:enum(decision|fact|pattern|task|observation) req :: node type
flag --tag:kv repeat :: metadata key:value pairs
ex mem node add "postgres 15 required" --type fact --tag project:mem
ex mem node add "use TOON for --agent-out bodies" --type decision --tag project:mem --tag spec:ahf

Notes

  • ah2 <tool> <command-path> — always the first line
  • use — the canonical invocation template, one line only
  • arg <name>:<type> req|opt :: <purpose> — positional argument with scalar type
  • flag --<name>:<type> req|opt|repeat [default=…] :: <purpose> — flag definition
  • enum(a|b|c) — closed set of valid values; agents should use exactly one of these
  • ex — a valid, tested example invocation. Omit if it might drift or fail.
  • Target: fewer than 150 tokens for AH2
03

Error with hint

Source command: mem node add "postgres 15 required" --agent-out

When a required flag is missing, the tool returns an err record with a machine-readable code, a hint with the direct correction, and the canonical use form — everything an agent needs to retry correctly without calling --help.

Raw AHF output

err missing_flag flag=--type
hint --type enum(decision|fact|pattern|task|observation)
use mem node add <text> --type TYPE [--tag K:V...]

Notes

  • err <code> [key=value…] — machine-readable error code with structured metadata
  • flag=--type identifies exactly which flag is missing — no prose parsing needed
  • hint gives the direct correction, including valid enum values
  • use gives the full canonical invocation so the agent can rebuild from scratch
  • Error output should never just say "run --help" — give the agent enough to retry directly
04

List results — --agent-out

Source command: mem node list --agent-out

The most common runtime pattern. An ok status line carries envelope metadata, followed by a TOON result body, followed by a next record pointing the agent at a useful follow-up action.

Raw AHF + TOON output

ok nodes count=8 shown=8 more=0
nodes[#8]{id,type,tags,text}:
  n_101,decision,"project:mem|arch",Use Cobra for the reference CLI implementation
  n_102,fact,"project:mem|db",Mock data lives in data.go; no real storage in the reference impl
  n_103,pattern,db|ops,Use connection pool max 20 in all environments
  n_104,fact,"project:mem|api",AHF ok/err envelope always precedes the TOON body
  n_105,decision,"project:mem|spec",TOON is the encoding for --agent-out result bodies
  n_106,task,"project:mem|todo",Write reference Go implementation for agent-help
  n_107,pattern,agent|cli,Always include next records for paginated results
  n_108,fact,"project:mem|spec",more? record is a pointer not a shell command
next inspect "mem search query <text> --agent-out"

Notes

  • ok is the AHF protocol status line — always first, always AHF
  • count= and more= are envelope metadata — always on the ok line
  • nodes[#8]{id,type,tags,text}: is the TOON block header — [#8] is the length marker
  • Rows are two-space indented per TOON spec
  • Values are quoted only when they contain commas or spaces
  • next is an AHF follow-up record — the exact command the agent should call next
  • TOON carries the data; AHF carries the protocol and follow-up
05

Single object — --agent-out

Source command: mem project status --agent-out

When the result is a single structured object rather than a list, TOON encodes it as a vertical key/value table. The ok line uses a singular noun kind and carries no list metadata.

Raw AHF + TOON output

ok project
project[#6]{key,value}:
  name,agent-help
  version,0.1.0
  status,draft
  owner,team-cli
  spec,AHF-RFC.md
  repo,github.com/Zate/agent-help
next nodes "mem node list --agent-out"

Notes

  • ok project — singular noun, no list metadata (count=, more=) needed for a single object
  • TOON key/value table: one row per field, two columns (key,value)
  • This is the canonical TOON pattern for structured settings or config objects
  • next gives the agent a natural follow-up action

When there's no structured body at all — a bare acknowledgement or scalar status — you can use AHF key-value lines directly after ok without a TOON block (see §12.4):

ok node
id n_109
type fact
tags spec:ahf
created 2026-05-12
text "toon-go library works"
next list "mem node list --agent-out"
06

Pagination + warning

Source command: mem node list --limit 3 --agent-out

Paginated results set more=1 on the ok line and include a cursor token. A warn record signals truncation non-fatally. Two next records give the agent both the continuation command and an alternative action.

Raw AHF + TOON output

ok nodes count=8 shown=3 more=1 cursor=c_n_104
warn truncated shown=3 total=8
nodes[#3]{id,type,tags,text}:
  n_101,decision,"project:mem|arch",Use Cobra for the reference CLI implementation
  n_102,fact,"project:mem|db",Mock data lives in data.go; no real storage in the reference impl
  n_103,pattern,db|ops,Use connection pool max 20 in all environments
next "mem node list --limit 3 --cursor c_n_104 --agent-out"
next inspect "mem search query <text> --agent-out"

Notes

  • more=1 on the ok line signals paginated/truncated output — always include a next continuation
  • count=8 total available; shown=3 rows in this response; cursor=c_n_104 is the page token
  • warn truncated is an AHF record — non-fatal, appears after ok, before the TOON body
  • The TOON body carries only the rows for this page — [#3] reflects the actual row count
  • Two next records: the unlabelled one is the continuation; the labelled one is an alternative action
  • The cursor= value from the ok envelope is passed through verbatim in the next command
  • An agent that sees more=1 with no next record should treat this as a producer error
