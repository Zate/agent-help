Agent Help Format

The exact wire-format contract for adding --agent-help and --agent-out to a CLI. Most people should start with the homepage or implementation guide.

Core idea

agent-help adds an agent-facing surface beside existing human and software surfaces:

human:    tool --help
software: tool --json
agent:    tool --agent-help
agent:    tool subcmd args --agent-out

--agent-help and the --agent-out envelope use AHF line records. Structured result bodies SHOULD use TOON.

Flags

Human help breadcrumb

LLM agent? Use --agent-help for token-optimized usage.

--agent-help

tool --agent-help
tool subcmd --agent-help
tool group subcmd --agent-help

Document trailing placement. You may accept leading placement for compatibility, but do not teach it as the primary form.

--agent-out

tool subcmd args --agent-out

Emit an AHF status envelope first, then TOON for structured data.

AHF line format

  • UTF-8 plain text; ASCII preferred.
  • One record per line.
  • First token is the record prefix.
  • Metadata uses key=value.
  • Purpose text uses :: .
  • _ means null, unknown, missing, or not applicable.
  • | separates short lists inside one field value.
  • Avoid colors, markdown tables, decorative headings, and prose paragraphs.
ok nodes count=8 shown=3 more=1 cursor=c_n_104
cmd node list [--limit int] :: list memory nodes
next inspect "tool show n_102 --agent-out"

Record prefixes

PrefixMeaningContext

oksuccess status--agent-out
errerror statusany agent-facing error
warnnon-fatal warning--agent-out
cmdcommand entryAH1
usecanonical invocationAH2 or error
argpositional argument definitionAH2
flagflag definitionAH2
exvalid example invocationAH2
hintdirect correctionafter err
nextfollow-up or continuation commandafter ok or err
more?pointer to AH2 detailAH1

Scalar types

Use str, int, num, bool, path, url, id, ts, date, dur, kv, and enum(a|b) in arg and flag records.

--agent-help

AH1 index

ah1 <tool> :: <purpose>
cmd <command-signature> :: <purpose>
more? <tool> <cmd> --agent-help

AH2 detail

ah2 <tool> <command-path>
use <canonical invocation>
arg <name>:<type> <req|opt> :: <purpose>
flag --<name>:<type> <req|opt|repeat> [default=<value>] :: <purpose>
ex <valid example invocation>

--agent-out

Successful output starts with ok. Structured bodies SHOULD be TOON.

ok issues count=2 more=0
issues[#2]{key,status,summary}:
  PROJ-1,open,Add agent-help
  PROJ-2,done,Document examples
next inspect "tool issue PROJ-1 --agent-out"

If there is no structured body, simple key-value AHF lines MAY follow ok.

Errors and follow-up

err invalid_enum flag=--type got=note
hint --type enum(decision|fact|pattern|task|observation)
use mem node add <text> --type TYPE [--tag K:V...]

Errors should let an agent retry without reading human --help. For pagination or truncation, include more=1 or truncated=1 and an exact next command.

Accuracy requirements

  • tool --help includes the breadcrumb.
  • tool --agent-help emits AH1.
  • Every AH1 cmd has AH2 output.
  • AH2 required args and flags match real validation.
  • Examples run successfully or are omitted.
  • --agent-out emits ok or err first.
  • Paginated or truncated output includes next.
  • Secrets and unnecessary personal data are redacted.

Examples

See the examples page for annotated output samples, or copy the raw files from examples/.
