§ 01

The missing third output mode.

CLIs already speak two languages. They speak prose to humans and JSON to scripts. Agents need something between those: dense, line-oriented, action-ready text exposed by the CLI itself.

01

Humans

--help

Prose, tables, color, examples. Optimized for reading.

$ tool search --help
Usage: tool search [OPTIONS] QUERY

  Search the knowledge graph for nodes
  matching QUERY across tags and content.

Options:
  --type   TEXT  filter by record type
  --limit  INT   max records returned
  --help         show this message and exit
≈ 410 tokens
02

Software

--json

Strict serialization for scripts and parsers.

{
  "command": "search",
  "args": [{
    "name": "query",
    "type": "string",
    "required": true
  }],
  "flags": [
    {"name": "--type",
     "type": "string"}
  ]
}
≈ 140 tokens
03

Agents

--agent-help / --agent-out

Compact, line-oriented, action-ready. Optimized for token cost and call-the-next-tool.

ah2 tool search :: search records
use tool search <query> [--type TYPE]
arg query : string :: search text
flag --type : string :: filter
flag --limit : int :: cap rows
ex tool search postgres --limit 5
≈ 38 tokens

Token counts are illustrative. The point is the order of magnitude: agent mode trims human prose without paying JSON’s punctuation tax.

§ 02

Two layers. One convention.

The convention has a thin help layer (AHF) and an optional runtime layer that delegates result bodies to TOON. Each format does its job; neither tries to do both.

Layer 1

AHF — Agent Help Format

The line-record spec for --agent-help output and the ok/err protocol envelope for --agent-out.

ah1 tool :: example data CLI
cmd search <query> [--type TYPE] [--limit int] :: search records
cmd show <id> :: display one record
cmd rm <id> :: delete a record
more? tool <cmd> --agent-help
  • ah1 / cmd / more? — index
  • ah2 / use / arg / flag / ex — detail
  • err / next / hint — recovery
Layer 2

TOON — result bodies

--agent-out emits an AHF status line, then a TOON result body. AHF wraps; TOON carries.

ok nodes count=2 more=0
nodes[#2]{id,type,tags,text}:
  n_102,fact,project:billing,postgres 15 required
  n_088,decision,project:billing,migrate read model
next inspect "tool show n_102 --agent-out"
  • ok / err — protocol envelope (AHF)
  • [#N]{...} — TOON length marker and header
  • next — the exact follow-up command

§ 03

How an agent finds your tool.

Four hops from nothing to calling the next command. The breadcrumb in normal help is the only entry point an agent needs.

  1. 1

    Breadcrumb in --help

    Agent runs tool --help and sees a one-line pointer.

    $ tool --help
Usage: tool [OPTIONS] COMMAND
…
LLM agent? Use --agent-help for token-optimized usage.
  2. 2

    Index with --agent-help

    Agent runs tool --agent-help and gets an AH1 command index.

    ah1 tool :: example data CLI
cmd search <query>
cmd show <id>
more? tool <cmd> --agent-help
  3. 3

    Detail per command

    Agent runs tool search --agent-help for AH2 invocation detail.

    ah2 tool search
use tool search <query> [--limit int]
arg query : string
flag --limit : int
  4. 4

    Invoke with --agent-out

    Agent calls the command. It gets a dense, structured result.

    ok nodes count=2 more=0
nodes[#2]{id,type,text}:
  n_102,fact,postgres 15 required
next inspect "tool show n_102"

§ 04

No runtime reading list.

The docs in this repo teach people and coding agents how to add the flags. They are not instructions an agent must preload before using your CLI.

Runtime path

  1. 1 Agent sees the breadcrumb in tool --help.
  2. 2 Agent runs tool --agent-help.
  3. 3 Agent follows cmd, use, arg, and flag records.
  4. 4 Agent asks for --agent-out when it wants compact structured results.

§ 05

Add it to your CLI.

Three small changes. Most frameworks support agent-help as a hidden global flag with a generator that walks the command tree.

  1. 01~1 min

    Add the breadcrumb

    One line in normal --help output.

    LLM agent? Use --agent-help for token-optimized usage.
  2. 02~1 hr

    Wire up --agent-help

    A trailing global flag that emits AH1 at root and AH2 per subcommand.

    $ tool --agent-help          # AH1 index
$ tool subcmd --agent-help   # AH2 detail
  3. 03~2 hrs

    Add --agent-out where it matters

    Wrap result-returning commands. AHF envelope first, then TOON body.

    $ tool search query --agent-out
