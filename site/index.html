agent-help

Build CLIs that agents can use natively.

A tiny command-line convention for discovery, invocation, and dense runtime output. No server, plugin, or middleware required for the basic path.

Read the spec Use with an agent 
human:    tool --help
software: tool --json
agent:    tool --agent-help
agent:    tool search query --agent-out

The missing third output mode

01

Humans

Use prose, tables, color, and examples.

--help
02

Software

Use strict serialization and parsers.

--json
03

Agents

Use compact, line-oriented, action-ready text.

--agent-help / --agent-out

How discovery works

1

Agent sees the breadcrumb in normal --help.

2

Agent runs tool --agent-help for an AH1 command index.

3

Agent runs tool subcmd --agent-help for AH2 invocation detail.

4

Agent calls --agent-out for dense structured results.

AHF — Agent Help Format

The line-record spec for --agent-help output: command index (AH1), command detail (AH2), errors, follow-up commands.

ah1 tool :: example CLI
cmd search <query> [--limit int] :: search
cmd show <id> :: show one record
more? tool <cmd> --agent-help

TOON — result bodies

--agent-out emits an AHF status line, then a TOON result body. Each format does its job.

ok nodes count=2 more=0
nodes[#2]{id,type,tags,text}:
  n_102,fact,project:billing,postgres 15 required
  n_088,decision,project:billing,migrate read model
next inspect "tool show n_102 --agent-out"
