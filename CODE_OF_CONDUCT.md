# Code of Conduct

agent-help is a technical specification project. Participation should stay focused, specific, and constructive.

## Expected Behavior

- Be respectful of different implementation constraints and tool ecosystems.
- Critique ideas with concrete reasoning, examples, and tradeoffs.
- Keep discussions about specs, code, docs, and interoperability.
- Assume good faith, but prioritize clarity over consensus-by-vagueness.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks.
- Dismissing feedback without technical reasoning.
- Derailing spec discussions with unrelated promotion or ecosystem arguments.
- Publishing private information or credentials.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and issues that violate this code of conduct. Repeated or severe violations may result in temporary or permanent participation limits.

Report concerns through the repository issue tracker if public discussion is appropriate, or contact the maintainers privately when needed.
